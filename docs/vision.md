# Unix Toolkit Vision

## Purpose

Unix Toolkit is an opinionated engineering toolkit for Unix and Linux administrators, \
with a strong focus on enterprise automation using Ansible.

The goal is not simply to provide scripts, but to provide a consistent, \
repeatable engineering experience that enables administrators to build automation projects quickly, \
safely and according to best practices.

The toolkit captures years of operational experience in enterprise environments and turns that knowledge into reusable tools.

---

## Philosophy

The toolkit is built around several guiding principles:

* Automate repetitive work.
* Standardise project structure.
* Reduce human error.
* Make good practices the default.
* Keep tools simple, readable and maintainable.
* Build small utilities that work well together.
* Optimise for real-world administration rather than demonstrations.

---

## Design Principles

Every utility should:

* Have a single clear purpose.
* Provide helpful output.
* Fail gracefully with meaningful messages.
* Follow consistent naming conventions.
* Reuse common library functions wherever possible.
* Be easy to extend without rewriting existing code.

---

## Target Platforms

Current primary platforms include:

* IBM Power Systems
* AIX
* Red Hat Enterprise Linux
* SUSE Linux Enterprise Server

The toolkit is designed to support enterprise workloads including:

* SAP
* Oracle Database
* IBM FlashSystem
* PowerVM
* Red Hat Ansible Automation Platform

---

## Long-Term Goals

The vision is to provide a complete engineering toolkit capable of:

* Bootstrapping new Unix control nodes.
* Creating standardised Ansible projects.
* Building reusable automation roles.
* Validating automation before execution.
* Simplifying day-to-day systems administration.
* Providing reusable utilities for enterprise infrastructure.

Ultimately, Unix Toolkit aims to become the daily companion for Unix and Linux engineers \ 
working in complex enterprise environments.

---

## Guiding Principle

> Good automation does not simply save time—it captures knowledge, improves consistency, \ 
and enables every engineer to work at their best.
